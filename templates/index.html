About

My current work centers around RESTful APIs and the web application front ends that consume them. I enjoy working on the wide range of aspects involved in applications backed by RESTful APIs and cloud-based services. Currently, I contribute to project bringing scientific models to private and public clouds. The scientific models range from simple calculators for pesticide risk assessment to processor heavy, parallel implemented models handling spatial and Big Data.

Being a "Full-stack" developer across multiple projects allows me to apply a range of skills to solve the problems at hand. My certification as a Red Had System Administrator is put to use daily managing our Linux-based cloud instances. My Python and Flask/Django experience builds the backbone of our web applications framework. The NoSQL and SQL database backends round out the applications, and the JavaScript frameworks beautify the whole process.

Current Employment

I currently develop RESTful APIs, web applications, and manage Linux cloud instances at the United States Environmental Protection Agency as an Oak Ridge Institute for Science and Education (ORISE) fellow. Our main product is a collection of scientific models hosted as web-based services known as the Quantitative Environmental Domain (QED). I manage the cloud servers, implement the RESTful APIs and their documentation, and design and develop the front end web applications. QED is mainly Python (Flask/Django) based, with JavaScript frameworks (jQuery, React, etc...) beautifying the front ends. Some of the scientific models are modified versions of legacy code (Fortran 95) wrapped behind RESTful services and others are full Python implementations.
